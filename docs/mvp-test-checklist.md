# MVP 테스트 체크리스트

## 1. 설치 테스트
- [ ] APK 설치가 정상 완료된다.
- [ ] 앱 실행이 정상 동작한다.
- [ ] Splash/초기 화면이 정상 표시된다.

## 2. PDF 선택 테스트
- [ ] PDF 선택 버튼이 동작한다.
- [ ] 파일 선택기가 `application/pdf` 필터로 열리는지 확인한다.
- [ ] 선택한 PDF 파일명이 화면에 표시된다.
- [ ] 권한 오류 발생 시 사용자 안내 문구가 표시된다.

## 3. 렌더링 테스트
- [ ] PDF 첫 페이지가 정상 표시된다.
- [ ] 한 번에 한 장만 표시된다.
- [ ] `ContentScale.Fit` 기준 표시가 유지된다.
- [ ] PDF 비율이 깨지지 않는다.
- [ ] 손상 PDF에서 오류 메시지가 정상 표시된다.

## 4. 페이지 이동 테스트
- [ ] 다음 버튼 반복 클릭 시 정상 이동한다.
- [ ] 이전 버튼 반복 클릭 시 정상 이동한다.
- [ ] 첫 페이지에서 이전 버튼이 비활성화된다.
- [ ] 마지막 페이지에서 다음 버튼이 비활성화된다.
- [ ] 빠른 연타 상황에서도 페이지 이동이 안정적이다.
- [ ] render race condition(늦은 렌더 덮어쓰기)이 발생하지 않는다.

## 5. 마지막 페이지 복원 테스트
- [ ] 중간 페이지로 이동한다.
- [ ] 앱 종료 후 재실행한다.
- [ ] 최근 문서 항목을 클릭한다.
- [ ] 마지막으로 읽던 페이지가 복원된다.

## 6. 최근 문서 테스트
- [ ] 최근 문서 목록이 정상 표시된다.
- [ ] 동일 URI 재열기 시 기존 항목이 갱신된다.
- [ ] 최근 문서가 최대 10개로 유지된다.
- [ ] 개별 삭제가 정상 동작한다.
- [ ] 전체 비우기가 정상 동작한다.
- [ ] 권한 만료/파일 삭제 문서는 열기 실패 시 자동 제거된다.

## 7. Zoom 테스트
- [ ] pinch 확대가 동작한다.
- [ ] pinch 축소가 동작한다.
- [ ] scale 범위가 1f~3f로 제한된다.
- [ ] 페이지 이동 시 scale이 1f로 초기화된다.
- [ ] 확대 후 고해상도 재렌더링이 적용된다.
- [ ] 빠른 zoom 동작 후에도 앱 멈춤이 없다.

## 8. 메모리/안정성 테스트
- [ ] PDF를 10회 반복 열어도 크래시가 없다.
- [ ] 페이지를 50회 이동해도 안정적이다.
- [ ] 앱 백그라운드/복귀 시 정상 복원된다.
- [ ] 화면 회전 시 크래시가 없다.
- [ ] 전체 테스트 동안 앱 크래시가 발생하지 않는다.
